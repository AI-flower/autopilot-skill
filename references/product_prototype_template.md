## 执行步骤（Phase 4 自动执行）

### Step 1：创建输出目录

```bash
OUTPUT_DIR="$HOME/Desktop/{产品名称}-设计-$(date +%Y%m%d-%H%M)"
mkdir -p "$OUTPUT_DIR"
```

### Step 2：生成信息架构

在生成代码前，先内部规划（不展示给用户）：
- 页面列表及每页的核心内容
- 导航结构（底部导航 / 顶部导航 / 侧边栏）
- 主色调 + 辅色 + 背景色

### Step 3：生成交互原型 HTML 文件

**每个页面一个 HTML 文件**，遵循以下规范：

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>{页面名称} - {产品名称}</title>
  <style>
    /* 内联 CSS，无需外部依赖 */
    /* 使用产品主色调 */
    /* 移动优先设计 */
  </style>
</head>
<body>
  <!-- 页面内容 -->
  <!-- 导航栏（与其他页面一致） -->
  <!-- 主体内容 -->
  <!-- 底部导航或返回按钮 -->
  <script>
    /* 页面跳转逻辑 */
    /* 简单交互效果 */
  </script>
</body>
</html>
```

**设计规范**：
- 使用内联 CSS，不依赖外部库（确保离线可用）
- 字体：系统默认字体栈（-apple-system, 'PingFang SC', sans-serif）
- 图片：用 CSS 渐变 / SVG 占位，不依赖外部图片链接
- 颜色：用 CSS 变量统一管理主色
- 所有按钮/链接都有对应的页面跳转

**必须包含的页面**（根据产品类型调整）：
- 首页 / 主界面（index.html）
- {功能页1}.html
- {功能页2}.html
- 详情页 / 内页（detail.html，如适用）
- 个人中心 / 设置页（profile.html，如适用）

## 常见产品类型参考

### App（移动端）
- 页面宽度：375px（iPhone 标准）
- 导航：底部 Tab Bar（4-5 项）
- 必含：启动页 / 首页 / 核心功能页 / 个人中心

### 管理后台
- 页面宽度：1280px 全宽
- 导航：左侧侧边栏
- 必含：仪表盘 / 列表页 / 详情页 / 表单页

### 落地页 / Landing Page
- 单页滚动设计
- 必含：Hero 区 / 功能介绍 / 用户案例 / CTA 按钮
- 响应式：桌面 + 移动端

### 电商 / 商城
- 必含：首页 / 分类页 / 商品详情 / 购物车 / 订单
- 元素：商品卡片 / 价格标签 / 加入购物车按钮

### 内容平台
- 必含：Feed 流 / 内容详情 / 搜索 / 个人主页
- 元素：卡片列表 / 标签 / 点赞评论

## 注意事项

1. **不要询问技术细节**：设计师不懂框架，统一用纯 HTML+CSS+JS
2. **不要用 CDN 链接**：假设无网络也能看，所有依赖内联
3. **颜色要有品味**：根据行业推断合适的主色，不要用默认蓝
4. **内容要真实感**：用符合产品调性的文案，不要用 Lorem ipsum
5. **图标用 Unicode / SVG**：不依赖 Font Awesome 等外部库
